# Conversion management

A conversion in Google Ads is when a user performs some specified action after clicking an ad or viewing a Display Network ad. Conversions represent customer interactions vital to your advertising goals, such as purchasing a product, installing a mobile app, or signing up for an email list.

Conversion tracking provides key insights into users' actions after viewing or clicking an ad. You can keep track of users who call, buy a product, install a mobile app, and more.

The Google Ads API lets you programmatically manage the end-to-end workflow for conversion management. The conversion management guide is divided into the following sections that include creating conversions, importing offline conversions, adjusting conversion values, monitoring the health of your conversions, and grouping conversion actions into conversion goals.

- **Conversion action categories** (.../docs/conversions/categories) — Guidance on how to identify which conversion action types align with your business objectives, and how they map to the Google Ads API.
- **Getting started** (.../docs/conversions/getting-started) — How to set your account up for conversion tracking, with a code example showing how to create a conversion action.
- **Manage offline conversions** (.../docs/conversions/upload-offline) — How to import offline conversions to measure real world transactions such as a qualified lead over the phone or in-office payment. Recommended: start with (or upgrade to) Enhanced Conversions for Leads, which uses first-party data (customer email or phone number) for the most accurate measurement.
- **Manage online conversions** (.../docs/conversions/upload-online) — Website conversion actions are measured using Google Tag, but you can use the API to improve accuracy/performance by supplementing existing conversion tags with first-party conversion data (email, name, home address, phone) using Enhanced Conversions for Web.
- **Manage call conversions** (.../docs/conversions/upload-calls) — Implementation details for importing call conversions.
- **Manage store sales conversions** (.../docs/conversions/upload-store-sales-transactions) — Implementation details for importing offline transactions.
- **Adjust conversions after import** (.../docs/conversions/upload-adjustments) — How to modify conversions after they've been imported.
- **Manage conversion value rules** (.../docs/conversions/conversion-value-rules) — Setting up rules that automatically modify a conversion's value if it meets user-defined criteria.
- **Custom conversion variables** (.../docs/conversions/conversion-custom-variables) — Associate additional info, as tags, to an imported conversion.
- **Conversion goals** (.../docs/conversions/goals/overview) — Organize sets of conversion actions to optimize towards advertising goals.
- **Monitor conversion import health** (.../docs/conversions/upload-summaries) — Retrieve offline data diagnostics to review the health of conversion import/adjustment processes.
- **Reporting** (.../docs/conversions/reporting) — How to report on conversion action performance, plus a mapping of Google Ads UI metrics to the API.
